## Module-Specific Permissions Summary

### Letters Management
- **Create Templates**: SUPER_ADMIN, ADMIN
- **Generate Letters**: SUPER_ADMIN, ADMIN, CAMPUS_ADMIN, DEAN, REGISTRY_ADMIN, REGISTRY, REGISTRY_ADMISSION
- **Sign Letters**: SUPER_ADMIN, ADMIN, CHANCELLOR, DEAN, HEAD_OF_DEPARTMENT, REGISTRY_ADMIN
- **Issue Letters**: SUPER_ADMIN, ADMIN, REGISTRY_ADMIN
- **View Own**: STUDENT, STAFF roles

### Business Center
- **Generate Pins**: SUPER_ADMIN, ADMIN, BUSINESS_CENTER, CAMPUS_BUSINESS_CENTER, FINANCE
- **Process Sales**: BUSINESS_CENTER, CAMPUS_BUSINESS_CENTER
- **Verify Pins**: BUSINESS_CENTER, CAMPUS_BUSINESS_CENTER, REGISTRY_ADMISSION, STUDENT (use)
- **Financial Reports**: SUPER_ADMIN, ADMIN, CHANCELLOR, FINANCE, FINANCE_SECRETARIAT, BUSINESS_CENTER

### Examinations
- **Schedule Exams**: SUPER_ADMIN, ADMIN, EXAMS, FACULTY_EXAM, DEAN
- **Assign Invigilators**: EXAMS, FACULTY_EXAM, HEAD_OF_DEPARTMENT
- **Enter Grades**: LECTURER, PART_TIME_LECTURER
- **Approve Grades**: DEAN, HEAD_OF_DEPARTMENT, REGISTRY_ACADEMIC
- **View Results**: Students (own), Parents (linked student), Academic staff (their courses)

### Finance
- **Create Fee Structure**: SUPER_ADMIN, ADMIN, FINANCE, FINANCE_SECRETARIAT
- **Process Payments**: FINANCE, FINANCE_STAFF, ACCOUNTANT, BUSINESS_CENTER, CAMPUS_BUSINESS_CENTER
- **Generate Reports**: SUPER_ADMIN, ADMIN, CHANCELLOR, FINANCE, FINANCE_SECRETARIAT, ACCOUNTANT
- **Reconciliation**: FINANCE, ACCOUNTANT

### Student Management
- **Add Students**: SUPER_ADMIN, ADMIN, REGISTRY_ADMIN, REGISTRY, REGISTRY_ADMISSION
- **Edit Records**: SUPER_ADMIN, ADMIN, REGISTRY_ADMIN, REGISTRY, REGISTRY_ACADEMIC
- **View Students**: Academic staff (their courses), Student Section, Student Warden (residence)
- **Student Welfare**: STUDENT_SECTION, STUDENT_SECTION_STAFF, STUDENT_WARDEN

### Communications
- **System-Wide**: SUPER_ADMIN, CHANCELLOR
- **Campus-Wide**: SUPER_ADMIN, ADMIN, CAMPUS_ADMIN
- **Faculty-Wide**: DEAN, FACULTY_ADMIN
- **Department**: HEAD_OF_DEPARTMENT
- **Course**: LECTURER, PART_TIME_LECTURER
- **Students**: STUDENT_SECTION, REGISTRY (official)

## Permission Hierarchy

```
Level 1: SUPER_ADMIN (All permissions)
├── Level 2: CHANCELLOR (Strategic oversight)
├── Level 2: ADMIN (Campus-wide admin)
│   ├── Level 3: CAMPUS_ADMIN (Single campus)
│   ├── Level 3: DEAN (Faculty)
│   │   └── Level 4: HEAD_OF_DEPARTMENT (Department)
│   │       └── Level 5: LECTURER, PART_TIME_LECTURER (Course)
│   └── Level 3: REGISTRY_ADMIN (Registry operations)
│       └── Level 4: REGISTRY, REGISTRY_ADMISSION, REGISTRY_HR, REGISTRY_ACADEMIC
├── Level 2: FINANCE_SECRETARIAT (System-wide finance)
│   └── Level 3: FINANCE (Campus finance)
│       └── Level 4: FINANCE_STAFF, ACCOUNTANT
├── Level 2: ELEARNING_ADMIN (eLearning platform)
├── Level 2: SPS_ADMIN (Professional Studies)
│   └── Level 3: SPS_STAFF
└── Level 2: EXAMS (University examinations)
    └── Level 3: FACULTY_EXAM (Faculty exams)

Support Services (Campus-level):
- BUSINESS_CENTER (System) / CAMPUS_BUSINESS_CENTER (Campus)
- LIBRARY, ID_CARD_PRINTING, HELP_DESK
- STUDENT_SECTION, STUDENT_WARDEN
- HUMAN_RESOURCES
- EXTRAMURAL_STUDIES

End Users:
- STUDENT, PARENT
```

## Implementation Notes

### Database Migration Required
After updating the User model with new roles, run:
```bash
python manage.py makemigrations authentication
python manage.py migrate authentication
```

### Permission Checking in Views
```python
from apps.authentication.models import User

def check_permission(user, required_roles):
    return user.role in required_roles

# Example
if check_permission(request.user, [User.FINANCE, User.FINANCE_STAFF, User.ACCOUNTANT]):
    # Allow financial operation
    pass
```
